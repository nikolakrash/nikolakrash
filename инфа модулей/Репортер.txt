МОДУЛЬ «РЕПОРТЁР» – ИНСТРУКЦИЯ ДЛЯ ПОЛЬЗОВАТЕЛЕЙ
------------------------------------------------

1. НАЗНАЧЕНИЕ МОДУЛЯ
Модуль «Репортёр» предназначен для массовой отправки жалоб (репортов) в Telegram:
- можно отправлять жалобы на каналы, чаты, пользователей, отдельные сообщения;
- можно использовать сразу несколько аккаунтов для ускорения;
- есть выбор причины жалобы, текста жалобы и задержек между отправками;
- формируется краткий отчёт по каждому запуску.

Используйте модуль аккуратно: злоупотребление жалобами может привести к ограничениям на ваших аккаунтах.


2. ЧТО МОЖНО РЕПОРТИТЬ
- Публичные и приватные каналы (`https://t.me/username`, `https://t.me/+hash`, `@username`);
- Чаты и супергруппы;
- Отдельные сообщения (ссылки вида `https://t.me/канал/ID` или `https://t.me/c/ID/MSG_ID`);
- Отдельные аккаунты по юзернейму (`@username`).

Каждый объект указывается **с новой строки** в поле «Список ссылок».


3. ОСНОВНЫЕ НАСТРОЙКИ

3.1. Список ссылок
- Вставьте все цели для жалоб в большое текстовое поле:
  - один адрес на строку;
  - допустимы форматы:
    - `https://t.me/username`
    - `https://t.me/+hash`
    - `https://t.me/channel/123`
    - `https://t.me/c/1234567890/1`
    - `@username`.

3.2. Причина жалобы
- Выберите **ровно одну** причину из списка:
  - Спам;
  - Насилие;
  - Порнография;
  - Контент для взрослых;
  - Нарушение авторских прав;
  - Географически нерелевантный;
  - Фейковые аккаунты/каналы;
  - Незаконные наркотики;
  - Личные данные;
  - Другое.

3.3. Сообщения (тексты жалоб)
- Поле «Сообщение (опционально)»:
  - на каждой строке можно указать **вариант текста жалобы**;
  - если строк несколько, модуль будет выбирать текст по выбранному режиму.

3.4. Режим выбора сообщения
- «По порядку (по кругу)»:
  - тексты жалоб берутся по очереди и «ходят» по кругу;
  - распределяются между аккаунтами и ссылками, чтобы не было одинаковых текстов подряд.
- «В рандомном порядке»:
  - для каждой жалобы выбирается случайный вариант текста.

3.5. Задержка от / до (секунд)
- «Задержка от» и «Задержка до» задают диапазон паузы между жалобами:
  - для каждой отправки случайно выбирается число от **минимума** до **максимума**;
  - помогает избежать FloodWait и лишних ограничений.

3.6. Аккаунты
- В правой части интерфейса:
  - список всех авторизованных аккаунтов;
  - можно фильтровать:
    - «Аккаунты без активных задач»;
    - «Аккаунты без ограничений».
- Отметьте те аккаунты, которые будут использоваться для отправки жалоб.
- Без выбранных аккаунтов модуль не запустится.


4. ПОРЯДОК РАБОТЫ (ШАГ ЗА ШАГОМ)
1) Откройте модуль «Репортёр».
2) Вставьте в поле «Список ссылок» все цели, на которые хотите отправить жалобы.
3) Выберите **одну** причину жалобы.
4) При необходимости:
   - пропишите один или несколько вариантов текста жалоб;
   - выберите режим выбора сообщения (по порядку / случайно).
5) Задайте задержки:
   - «Задержка от (сек)» – минимальная пауза;
   - «Задержка до (сек)» – максимальная пауза.
6) В блоке «Активные аккаунты» выберите один или несколько аккаунтов для работы.
7) Нажмите **«Запустить репортёр»**.
8) Следите:
   - за блоком «Статистика» – количество успешных/ошибочных жалоб, текущая ссылка и аккаунт;
   - за блоком «Логи операций» – детальные события (какие ссылки, какие аккаунты, какие ошибки).
9) При необходимости нажмите **«Остановить»**, чтобы модуль завершил работу как можно быстрее и корректно.


5. ЛОГИ, СТАТИСТИКА И ОТЧЁТЫ
- **Статистика**:
  - показывает:
    - число успешных жалоб;
    - число ошибок;
    - сколько ссылок обработано из общего списка;
    - какая ссылка и какой аккаунт обрабатываются сейчас;
    - сколько времени идёт текущий запуск;
    - краткий статус (запущен, остановлен, ошибка).
- **Логи операций (UI‑логи)**:
  - отдельный блок под статистикой;
  - показывает пошаговый ход работы:
    - запуск модуля;
    - назначение задачи на аккаунты;
    - отправка жалоб по конкретным ссылкам;
    - ошибки доступа или лимитов;
    - завершение/остановку.
- **История запусков**:
  - внизу страницы показывается таблица последних запусков:
    - время старта;
    - сколько ссылок обработано;
    - сколько жалоб прошло успешно и сколько с ошибкой;
    - длительность запуска;
    - было ли принудительное прекращение.
- **Отчёты**:
  - при завершении запуска создаётся запись в истории;
  - в блоке «Отчёт» доступна ссылка для скачивания `.txt` файла с подробным списком ссылок, аккаунтов и статусов.


6. РЕКОМЕНДАЦИИ ПО БЕЗОПАСНОСТИ
- Не используйте слишком маленькие задержки (0–1 секунда) при большом количестве ссылок и аккаунтов.
- Не запускайте одновременно много агрессивных модулей (рассылки, инвайты, репортёр) на одних и тех же аккаунтах.
- Следите за логами:
  - если аккаунт постоянно выдаёт ошибки, снимите его из списка и проверьте отдельно;
  - при частых FloodWait стоит увеличить задержки или временно снизить нагрузку.
- Соблюдайте правила Telegram и не используйте репортёр для рейда по невиновным каналам/пользователям – это может привести к блокировке аккаунтов.


7. ПАРАЛЛЕЛЬНАЯ РАБОТА
- «Репортёр» может работать параллельно с другими модулями, но:
  - чем больше одновременно активных задач на одном аккаунте, тем выше риск ограничений;
  - лучше распределять нагрузку по разным аккаунтам и не перегружать один и тот же номер.
